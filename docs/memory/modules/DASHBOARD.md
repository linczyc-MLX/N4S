# Module: Dashboard

> **Status**: ✅ Live | **Header**: #1e3a5f | **Last Updated**: 2026-02-22

## Purpose
Landing screen after login. Project selection, creation, and high-level progress overview across all modules.

## Key Files
- `src/App.jsx` — Module switching, project routing, docs overlay
- `src/contexts/AppContext.jsx` — Master context: project state load/save, all module data
- `src/contexts/AuthContext.jsx` — Authentication state

## Key Patterns
- Project data stored in single `project_data` JSONB row per project in MySQL
- AppContext loads all module data on project switch, provides to children
- Module progress gates: A–E. Gate D = questionnaireCompletedAt. Gate C = moduleReviewStatus (8 modules).
- Documentation overlay: 4-tab system (Overview/Workflow/Gates/Reference), controlled by `showDocs`/`onCloseDocs` props from App.jsx. `modulesWithDocs` array controls which modules show the docs button.

## Gotchas
- Project switching triggers full data reload — ensure no stale state between projects
- Static PDF docs in `public/docs/` generated by ReportLab Python scripts (run locally, not on server)
